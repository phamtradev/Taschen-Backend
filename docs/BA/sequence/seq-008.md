sequenceDiagram

    participant DB as 💾 MySQL DB

    Note over WS,DB: === STEP 1: Warehouse creates disposal request ===
    WS->>DRC: POST /api/disposal-requests
    DRC->>DRS: createDisposalRequest(request)

    NS-->>AD: 🔔 New disposal request!
    DRS-->>DRC: DisposalRequestResponse
    DRC-->>WS: 201 Created + DisposalRequestResponse

    Note over WS,DB: === STEP 2: Admin reviews ===
    AD->>DRC: PUT /api/disposal-requests/{id}/approve
    DRC->>DRS: approveDisposalRequest(id)
    DRS->>DRR: findById(id)
    DRR->>DB: SELECT * FROM disposal_requests WHERE id = ?
    DB-->>DRR: DisposalRequest
    DRR-->>DRS: DisposalRequest
    DRS->>DB: SELECT * FROM disposal_request_items WHERE disposal_request_id = ?
    DB-->>DRS: List<DisposalRequestItem>
    loop For each DisposalRequestItem
        DRS->>BR: findById(batchId)
        BR->>DB: SELECT * FROM batches WHERE id = ?
        DB-->>BR: Batch
        BR-->>DRS: Batch
        DRS->>DRS: Validate: item.quantity <= batch.remainingQuantity
        DRS->>DB: UPDATE batches SET remaining_quantity = remaining_quantity - ?<br/>WHERE id = ?
        DRS->>DB: UPDATE disposal_request_items<br/>SET remaining_quantity_after = remaining_quantity - item.quantity
    end
    DRS->>DB: UPDATE disposal_requests SET status='APPROVED', processed_at=NOW()
    DRS->>NS: notifyWarehouseStaff("Disposal approved")
    NS-->>WS: 🔔 Disposal approved!
    DRS->>BS: syncStockToBook(bookId)
    BS->>DB: SELECT SUM(remaining_quantity) FROM batches WHERE book_id = ?
    DB-->>BS: totalStock
    BS->>DB: UPDATE books SET stock_quantity = ? WHERE id = ?
    DRS-->>DRC: DisposalRequestResponse
    DRC-->>AD: 200 OK + DisposalRequestResponse

    Note over WS,DB: === Rollback scenario ===
    alt REJECT

        DRC->>DRS: rejectDisposalRequest(id)
        DRS->>DB: UPDATE disposal_requests SET status='REJECTED'
        DRS->>NS: notifyWarehouseStaff("Disposal rejected")
        NS-->>WS: 🔔 Disposal rejected!
        DRS-->>DRC: DisposalRequestResponse
        DRC-->>AD: 200 OK
