## Customer Return Request Flow

    participant RRS as 📋 ReturnRequestService

    participant NS as 🔔 NotificationService
    participant SE as 👤 Seller
    participant DB as 💾 MySQL DB

    Note over U,DB: === STEP 1: Customer creates return request ===
    U->>RRC: POST /api/return-requests
    RRC->>RRS: createReturnRequest(request)
    RRS->>OR: findById(orderId)
    OR->>DB: SELECT * FROM orders WHERE id = ?
    DB-->>OR: Order
    OR-->>RRS: Order
    RRS->>RRS: Validate order belongs to user
    RRS->>RRR: save(returnRequest)
    RRR->>DB: INSERT INTO return_requests (status=PENDING)
    RRS->>NS: notifySeller("New return request")
    NS-->>SE: 🔔 New return request!
    RRS-->>RRC: ReturnRequestResponse
    RRC-->>U: 201 Created + ReturnRequestResponse

    Note over U,DB: === STEP 2: Seller approves/rejects ===
    SE->>RRC: PUT /api/return-requests/{id}/approve
    RRC->>RRS: approveReturnRequest(id)
    RRS->>RRR: findById(id)
    RRR->>DB: SELECT * FROM return_requests WHERE id = ?
    DB-->>RRR: ReturnRequest
    RRR-->>RRS: ReturnRequest
    RRS->>DB: UPDATE return_requests SET status='APPROVED'
    RRS->>DB: UPDATE orders SET status='RETURNED' WHERE id = ?
    RRS->>NS: notifyCustomer("Return approved")
    NS-->>U: 🔔 Return approved!
    RRS-->>RRC: ReturnRequestResponse
    RRC-->>SE: 200 OK + ReturnRequestResponse

    alt REJECT
        SE->>RRC: PUT /api/return-requests/{id}/reject
        RRC->>RRS: rejectReturnRequest(id)
        RRS->>DB: UPDATE return_requests SET status='REJECTED'
        RRS->>NS: notifyCustomer("Return rejected")
        NS-->>U: 🔔 Return rejected!
        RRS-->>RRC: ReturnRequestResponse
        RRC-->>SE: 200 OK
    end
