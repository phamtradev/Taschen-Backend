## 1. Mô tả

Cho phép User tạo đơn hàng mới từ thông tin địa chỉ giao hàng và mã khuyến mãi (nếu có).

## 2. Basic Flow

| Step | Actor | System | Action |
|------|-------|--------|--------|
| 1 | User | | Gửi `POST /api/orders` với thông tin đơn hàng |
| 2 | | OrderController | Validate request |
| 3 | | OrderService | Kiểm tra Address hợp lệ |
| 4 | | | Áp dụng Promotion nếu có mã |

| 8 | | | Trừ stock từ Batch (FIFO) |
| 9 | | | Trả về `OrderResponse` |
| 10 | User | | Nhận order, chọn phương thức thanh toán |

## 3. API Endpoint

```
POST /api/orders
Body: {
  "addressId": 1,
  "promotionCode": "SUMMER2026"
}
Auth: Cần đăng nhập
```

## 4. Alternative Flows

### 4.1 Invalid Address
- Khi address không tồn tại hoặc không thuộc user:
  - Trả về HTTP 400 "Invalid address"

### 4.2 Invalid Promotion Code
- Khi promotion code không hợp lệ:
  - Trả về HTTP 400 "Invalid promotion code"

### 4.3 Out of Stock
- Khi không đủ stock:
  - Trả về HTTP 400 "Insufficient stock"
