## 1. Mô tả

Cho phép User thanh toán đơn hàng qua cổng thanh toán VNPay.

## 2. Basic Flow

### 2.1 Create Payment URL

| Step | Actor | System | Action |
|------|-------|--------|--------|
| 1 | User | | Gửi `POST /api/payments/vnpay/create/{orderId}` |

| 3 | | VnPayService | Tạo VNPay payment URL với signature |
| 4 | | | Trả về `paymentUrl` |

## 3. API Endpoints

```
POST /api/payments/vnpay/create/{orderId}
GET  /api/payments/vnpay/return
Auth: Cần đăng nhập
```

---

## 4. Alternative Flows

### 4.1 Payment Failed
- Khi VNPay trả về mã lỗi:
  - Giữ nguyên status = UNPAID
  - Trả về thông báo lỗi cho user

### 4.2 Invalid Signature
- Khi signature không hợp lệ:
  - Trả về HTTP 400 "Invalid signature"
