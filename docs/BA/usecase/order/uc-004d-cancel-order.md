# UC-004d: Cancel Order

## 1. Mô tả

Cho phép User hủy đơn hàng chưa được xử lý. Đơn hàng chỉ có thể hủy khi status là UNPAID hoặc PENDING.

---

## 2. Basic Flow

| Step | Actor | System | Action |
|------|-------|--------|--------|
| 1 | User | | Gửi `PUT /api/orders/{orderId}/cancel` |
| 2 | | OrderController | Gọi `orderService.cancelOrderByCustomer()` |
| 3 | | OrderService | Kiểm tra order thuộc user |
| 4 | | | Kiểm tra status cho phép hủy (UNPAID, PENDING) |
| 5 | | | Hoàn lại stock vào Batch |
| 6 | | | Đổi status → CANCELLED |
| 7 | | | Gửi notification cho user |
| 8 | | | Trả về updated OrderResponse |
| 9 | User | | Nhận xác nhận hủy |

PUT /api/orders/{orderId}/cancel

---

## 4. Alternative Flows

### 4.1 Order Already Processing
- Khi cố hủy order đang ở trạng thái PROCESSING hoặc DELIVERING:
  - Trả về HTTP 400 "Cannot cancel order in current status"

### 4.2 Order Not Found
- Khi orderId không tồn tại:
  - Trả về HTTP 404 "Order not found"

### 4.3 Unauthorized Cancel
- Khi order không thuộc user đang login:
  - Trả về HTTP 403 "Access denied"
