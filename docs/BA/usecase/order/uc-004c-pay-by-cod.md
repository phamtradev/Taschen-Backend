# UC-004c: Pay by COD

> **Use Case ID:** UC-004c

## 1. Mô tả

Cho phép User chọn thanh toán khi nhận hàng (COD). Đơn hàng sẽ được chuyển sang trạng thái PENDING.

---

## 2. Basic Flow

| Step | Actor | System | Action |
|------|-------|--------|--------|
| 1 | User | | Gửi `PUT /api/orders/{orderId}/pay-cod` |
| 2 | | OrderController | Gọi `orderService.payByCOD()` |
| 3 | | OrderService | Cập nhật `paymentMethod = COD` |
| 4 | | | Đổi status → PENDING |
| 5 | | | Trả về updated OrderResponse |
| 6 | User | | Nhận xác nhận đơn hàng |

---

## 3. API Endpoint

```
PUT /api/orders/{orderId}/pay-cod
Auth: Cần đăng nhập
```

---

## 4. Alternative Flows

### 4.1 Order Not Found
- Khi orderId không tồn tại:
  - Trả về HTTP 404 "Order not found"

### 4.2 Order Already Paid
- Khi order đã thanh toán:
  - Trả về HTTP 400 "Order already paid"

### 4.3 Order Cancelled
- Khi order đã bị hủy:
  - Trả về HTTP 400 "Cannot pay for cancelled order"
